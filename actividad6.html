Actividad 6: Investigación NAS (Sistemas Operativos, Backups y FTPS)

“Los dispositivos NAS se han consolidado como una solución clave en entornos domésticos, PYMES y corporativos…”

1. Introducción

Un NAS (Network Attached Storage) es un dispositivo de almacenamiento conectado a la red que permite compartir archivos, gestionar usuarios, realizar copias de seguridad y ejecutar servicios adicionales.

2. Sistemas Operativos para NAS

2.1 Factores clave

  • Interfaz de usuario
  • Nivel técnico requerido
  • Funciones adicionales (Docker, VMs, VPN…)
  • Seguridad
  • Compatibilidad de hardware

2.2 Principales sistemas operativos

TrueNAS (Core y SCALE)

Basado en ZFS, muy robusto, ideal para entornos profesionales.

Synology DSM

Interfaz muy intuitiva, ecosistema de apps propio.

QNAP QTS / QuTS hero

Muy completo, con soporte para virtualización y contenedores.

Unraid

Flexible, permite mezclar discos de diferentes tamaños.

OpenMediaVault

Gratuito, ideal para hardware reciclado.

3. Copias de Seguridad en NAS

3.1 Regla 3-2-1

  • 3 copias de los datos
  • 2 tipos de almacenamiento
  • 1 copia fuera del sitio

3.2 Tipos de backup

  • Completo
  • Incremental
  • Diferencial
  • Snapshots
  • Completo sintético

3.3 Métodos de backup

  • NAS a NAS (rsync, SMB)
  • Backup en la nube (S3, Azure, Backblaze)
  • Backup USB
  • Backup desde PC (Time Machine, Historial de archivos…)

3.4 Herramientas nativas

  • Synology Hyper Backup
  • QNAP HBS
  • ASUSTOR Backup Plan
  • Rsync
  • Veeam Agent

3.5 Protección frente a ransomware

  • Snapshots automáticos
  • Permisos restringidos
  • Backups offline
  • Actualizaciones frecuentes
  • Bloqueo por intentos fallidos

4. Aplicaciones FTPS para subir archivos al NAS

4.1 FTPS vs FTP vs SFTP

  • FTP → sin cifrado
  • FTPS → FTP + SSL/TLS
  • SFTP → basado en SSH

4.2 Clientes FTPS

  • FileZilla
  • WinSCP
  • Cyberduck
  • CloudMounter
  • Classic FTP

4.3 Configuración FTPS en un NAS

  1. Activar servicio FTP
  2. Habilitar FTPS
  3. Configurar certificado SSL
  4. Abrir puertos
  5. Crear usuarios

4.4 Recomendaciones de seguridad

  • Usar FTPS o SFTP siempre
  • Certificados válidos
  • Restringir IPs
  • 2FA
  • Auditoría de accesos
